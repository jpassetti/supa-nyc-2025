Step 4: 2D Perlin Noise Island Map

Code is available on the Github repository.

Explore Perlin noise in two dimensions!
This step creates a 100x100 grid of blocks on a 1000x1000px canvas. Each block uses Perlin noise to decide if it's ocean, sand, land, or mountain.

How it works:

  • The canvas is filled with ocean blue by default.
  • For each block, we use Perlin noise to get a value between 0 and 1.
  • If the value is 0.4 or less, it's ocean (left as background).
  • If the value is above 0.4 up to 0.6, we draw a sand block (beach).
  • If the value is above 0.6 up to 0.8, we draw a land block (dirt color).
  • If the value is above 0.8, we draw a mountain block (gray).

What is 2D Perlin noise?
Perlin noise is a type of smooth, natural-looking randomness often used in computer graphics and games. 2D Perlin noise means you provide two coordinates (x, y) and get a value between 0 and 1. Nearby points have similar values, so you get smooth "islands" and "continents" instead of random speckles. This is why it's great for making terrain, clouds, and textures that look organic!
Learn more: p5.js noise() reference
  • Each block in the grid uses its (x, y) position as input to the noise function: noise(x, y).
  • Blocks that are close together have similar noise values, so you get smooth transitions between ocean, sand, land, and mountain.
  • Changing the scale of the input (multiplying x and y by a small number) controls how "zoomed in" or "zoomed out" the terrain looks.

Classroom/Teacher Notes:
  • Discuss how 2D Perlin noise can be used for procedural map generation.
  • Ocean blocks are now n ≤ 0.4, sand is 0.4 < n ≤ 0.6, land is 0.6 < n ≤ 0.8, and mountain is n > 0.8.
  • Challenge students to change the thresholds or colors for different effects.
  • Try animating the noise by changing the noise seed or offset over time.
