Step 1: Tile Grid with Random Black Shapes (Processing.py)

Code is available on the Github repository.

Create a grid of tiles, each with a random black shape!
This step matches the JavaScript demo, but uses Python code in Processing's Python mode.

Python code for this step:
py/step-01.py

How it works:

  • Uses two nested for loops to create a grid of tiles (12x12 by default).
  • Each tile randomly draws a black circle, square, or line (all the same size, no rotation).
  • Randomness is used to select the shape for each tile, so every run is different.
  • All drawing is done in the setup() function, so the grid is static (not animated).
  • Great for discussing iteration, randomness, and visual constraints in creative coding!

Code Walkthrough:
  • Grid setup: The grid size is set with gridSize. Each cell's width/height is calculated from the canvas size.
  • Random shape selection: For each cell, random(3) picks 0, 1, or 2 (circle, square, or line).
  • Drawing: ellipse() draws a circle, rect() draws a square, and line() draws a horizontal line, all centered in the cell.
  • Remix: Try changing the grid size, adding color, or introducing new shapes for creative exploration.

Instructions:
  • Open py/step-01.py in the Processing IDE (Python mode).
  • Click the Run button (triangle) to see the grid composition.
  • Try changing the grid size, shape types, or add color for creative remixing!

Teacher Notes:
  • This sketch is a direct translation of the p5.js version for easy comparison.
  • Use this to discuss iteration, randomness, and constraints in art and design.
  • Ask students to predict what will happen if you change the grid size or shape selection logic.
  • Challenge students to remix: add color, change grid size, or add new shapes (triangles, arcs, etc).
  • Discuss how randomness can be used for both art and science, and how constraints shape creativity.

Remix Ideas:
  • Change the grid size or canvas size for different effects.
  • Add color, transparency, or gradients to the shapes.
  • Animate the grid by moving or rotating shapes in draw().
  • Let students invent their own rules for shape selection or placement.
